## Reading

Robust viability means at least two-thirds of seeds remain viable. The summary reports robust load levels explicitly because viability can be non-monotone: resource scaling changes routing, not only capacity, and can create phase islands rather than a single critical-load threshold. The shared randomization key makes baseline and policy comparisons paired within each seed.
